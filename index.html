🔁 זרימות מרכזיות

KYC / אימות נהג

  1. נהג מתחבר → Odin (S4)
  2. Odin שולח ל-Persona (app.withpersona.com)
  3. Persona מחזיר Webhook עם תוצאת אימות
  4. Odin שולח POST /webhooks/odin → S1 ו-S2

JWT Auth

  1. Odin מנפיק RS256 JWT token
  2. S1 ו-S2 מאמתים עם המפתח הציבורי
  3. אין צורך לגעת ב-S4 לכל בקשה

Vault — ארנק Taz

  1. אחרי אישור KYC → S4 ו-S1
  2. שולחים POST /wallets/create ל-S5
  3. S5 יוצר ארנק עם X-Vault-Internal-Key
  4. Background worker ב-S1 מנסה שוב אם נכשל

Dispatch נסיעה

  1. נוסע מזמין נסיעה → S1
  2. S1 שולח event הקצאה → S3
  3. S3 מפיץ לנהגים דרך Meta WhatsApp
  4. נהג מאשר → S3 → S1

LocalBiz — יצירת אתר

  1. עסק נרשם ב-tazo-web.com (S2)
  2. AI agents (LLM) יוצרים תוכן + אתר
  3. Google Maps לנתוני מיקום
  4. Green Invoice / Morning לתשלום

WhatsApp Onboarding

  1. נהג שולח הודעה למספר העסקי
  2. Odin (S4) מקבל Webhook מ-Meta
  3. שולח OTP ומדריך את הנהג דרך KYC
  4. Twilio כגיבוי אם Meta לא זמין
